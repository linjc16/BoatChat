登录界面具体操作见报告。其中，有很多try catch维护，如果出现卡住的问题不代表是程序BUG。

文件作用：

1.ChatWindow.xaml.cs为聊天界面。
2.P2PModule.cs为P2P模块
3.ServerConnct.cs为连接服务器 2,3两项代码编写时有参考官方MSDN文档
4.Window1.xamls，没有用的测试文件
5.Simple Styles.xmal 没有用的文件
6.MainWindow.xaml.cs为登录界面

界面编写参考了materialDesign中的Github中的教程。本次界面均有代码编写(.xaml)。

除此之外，鲁棒性可能没有做到极致，即可能还会出现意想不到的小Bug。

此外，图片加载设定的是绝对路径，即默认保存的路径(当前文件夹内的Image文件夹)，故在您的电脑上打开时，可能不会
加载图片。这一点模仿微信中，若文件的位置发生改变，则文件不存在。
测试时，只需要重新发送一个动态表情即可显示。


本程序只测试了在两台电脑上，在多台电脑上没有进行测试，但或许应该不会出现BUG.

此外：群聊、语音、公众号只做了控件
